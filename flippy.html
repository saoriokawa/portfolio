FLIPPY

UX Design

Project Overview

Flippy is a flashcard app to help learning vacabulary of new languages. Users can create their own set of flashcards or use a set created by another user to study vocabulary. It allows users to study vocabulary through different modes, such as spell quiz and test.

My Role

UX Designer, UI Designer

Tools

Balsamiq, Sketch, InVision, FlowMapp

Design Process

1. Competitive Analysis

First of all, competitive analysis of the three apps below were carried out.

Brainscape



Quizlet



Tinycards



2. User Interviews

After getting a basic idea of the current market, user interviews were conducted to find out problems that users have. The interview results revealed that users face some common problems, such as

  • Time/Space to write down vocabulary to memorise it
  • A way to effectively learn articles for each noun
  • Variation in vocabulary learning process

3. Personas

Based on the results of the interviews, I created a user persona to make an image of target users clearer.

4. User Flow

The persona, Cathy, revealed that users would use the app to deal with the following challenges;

  • Cathy doesn't have time to write down vocabulary
  • Cathy finds it difficult to memorise articles for each noun
  • Cathy loses motivation when the learning process becomes monotone

With these challenges in mind, user flows were established as a summary of tasks that users would complete.

5. Sitemaps

Using the user flows as a basic idea, sitemaps were established to visualise the entire structure of the app.

6. Wireframes

Based on the user flows and the sitemaps above, wireframes were created. The process started from low-fidelity wireframes with a pen and paper, followed by mid-fidelity wireframes created using Balsamiq. Finally, high-fidelity wireframes were made utilising Sketch.
